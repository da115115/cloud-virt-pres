ZeroMQ

The NoSQL of messaging

To fix the world, we needed to do two things. One, to solve the general problem of "how to connect any code to any code, anywhere". Two, to wrap that up in the simplest possible building blocks that people could understand and use easily.

  • LGPL networking library
  • Socket-oriented
  • Message-based
  • High-level API
  • Developed by iMatix

Languages support

  • C/C++
  • Java
  • C#
  • Python
  • Ruby
  • Scala
  • Smalltalk
  • PHP
  • Objective-C
  • JavaScript
  • Haskell
  • Erlang
  • ...

Simple APIs

context = ZMQ::Context.new 1

requester = context.socket ZMQ::REQ
requester.connect "tcp://localhost:5555"

requester.send_string "Hello"
reply = ''
requester.recv_string reply

APIs are transport-agnostic

  • Local in-process: inproc
  • Local inter-process: ipc
  • Unicast: tcp
  • Multicast: pgm

Messaging patterns

  • request-reply
  • publish-subscribe
  • pipeline
  • exclusive pair

Concurrency

  • Thread-safe library and FFI extension
  • Managed threads pool for async I/O
  • No lock, no semaphore in application code

Robustness

  • Automatic reconnection
  • Monitored message queues
  • Smart network error handling

Awesome when coupled with a portable serialization format

  • BERT
  • MessagePack
  • Protocol Buffers
  • Avro
  • Thrift

Support

  • LGPL licence
  • Commercial support from iMatix
